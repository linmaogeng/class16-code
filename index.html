Swing Exercise

Overview

The main objective of this exercise is to allow you to write some 'useful' Swing code.

The Project

You'll be implementing a calculator, but one with a small twist: it calculates in base 4 (quaternary) only. (Wikipedia has way more information than you need.) So the only digits are 0, 1, 2, and 3. And 2 + 2 is not 4, but 10.

To keep things simple, division will be integer division ("decimal digits" in other bases get tricky). So in base 10, 37 / 4 = 9, right? In quaternary, we'd write that as 211 / 10 = 21.

In the skeleton, I've already set you up with some buttons. I've also given you a skeleton of a class that will do the actual calculation. Your big task is to think through the event handling and calculational logic. (Note that you can never have a "purely" GUI application—there always has to be some "business logic," too.)

Feel free to collaborate with teammates on this one. Nothing needs to be handed in for this exercise, but the next project might be based on this in some way.

Of course, you can get this working in any order you like. Maybe you want to get the button-clicks "working" first before you tackle the calculating logic. Maybe you want to get the calculator working in base 10 before you tackle the base 4 thing. Up to you...
